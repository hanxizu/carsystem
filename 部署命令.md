# GitHub 部署完整命令

## 一、初始化 Git 仓库并推送到 GitHub

### 1. 初始化本地仓库

```bash
# 进入项目目录
cd car-insurance-system

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 创建 .gitignore 文件（如果还没有）
cat > .gitignore << 'EOF'
node_modules/
dist/
.env
*.log
.DS_Store
*.tmp
.vercel
.railway
EOF

# 提交代码
git commit -m "Initial commit: 车险自动报价分发系统"
```

### 2. 在 GitHub 上创建仓库

**方式一：使用 GitHub CLI（推荐）**
```bash
# 安装 GitHub CLI（如果没有）
# macOS: brew install gh
# Windows: winget install --id GitHub.cli
# Linux: sudo apt install gh

# 登录 GitHub
gh auth login

# 创建仓库并推送
gh repo create car-insurance-system --public --source=. --remote=origin --push
```

**方式二：手动创建（网页操作）**

1. 访问 https://github.com/new
2. 仓库名称：`car-insurance-system`
3. 描述：车险自动报价分发系统
4. 选择 Public 或 Private
5. 不要勾选 "Add a README file"
6. 点击 "Create repository"

然后执行以下命令：
```bash
# 添加远程仓库（替换 YOUR_USERNAME 为你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/car-insurance-system.git

# 推送代码
git branch -M main
git push -u origin main
```

## 二、完整部署流程（从零开始）

### 一键部署脚本

创建 `deploy.sh` 文件：

```bash
#!/bin/bash

# 车险系统一键部署脚本

echo "🚀 开始部署车险自动报价分发系统"

# 1. 初始化 Git
echo "📦 初始化 Git 仓库..."
git init

# 2. 创建 .gitignore
cat > .gitignore << 'EOF'
node_modules/
dist/
.env
*.log
.DS_Store
*.tmp
.vercel
.railway
coverage/
.vscode/
.idea/
EOF

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "Initial commit: 自动报价分发系统"

# 5. 创建 GitHub 仓库（需要 gh CLI）
if command -v gh &> /dev/null; then
    echo "🔗 创建 GitHub 仓库..."
    gh repo create carsystem --public --source=. --remote=origin --push
else
    echo "⚠️  请手动创建 GitHub 仓库并执行以下命令："
    echo "git remote add origin https://github.com/hanxizu/carsystem.git"
    echo "git branch -M main"
    echo "git push -u origin main"
fi

echo "✅ 部署完成！"
```

运行脚本：
```bash
chmod +x deploy.sh
./deploy.sh
```

## 三、Git 常用命令速查

### 基础操作
```bash
# 查看状态
git status

# 查看分支
git branch

# 切换分支
git checkout -b feature/new-feature

# 添加文件
git add src/           # 添加整个目录
git add .              # 添加所有文件
git add --all          # 添加所有更改

# 提交
git commit -m "feat: 添加新功能"

# 推送
git push origin main

# 拉取
git pull origin main
```

### 分支管理
```bash
# 创建并切换到新分支
git checkout -b develop

# 合并分支
git checkout main
git merge develop

# 删除分支
git branch -d develop

# 查看所有分支
git branch -a
```

### 撤销操作
```bash
# 撤销暂存
git reset HEAD file.txt

# 撤销提交（保留修改）
git reset --soft HEAD~1

# 撤销提交（删除修改）
git reset --hard HEAD~1

# 修改最后一次提交
git commit --amend -m "新的提交信息"
```

## 四、部署到 Vercel（连接 GitHub）

### 方式一：Vercel CLI
```bash
# 安装 Vercel CLI
npm i -g vercel

# 部署（会自动连接 GitHub）
vercel --prod

# 后续更新只需
vercel --prod
```

### 方式二：Vercel 网页（推荐）

1. 访问 https://vercel.com
2. 点击 "Add New" → "Project"
3. 选择 GitHub 仓库 `car-insurance-system`
4. 配置：
   - Framework Preset: `Other`
   - Build Command: `npm run build`
   - Output Directory: `dist`
   - Install Command: `npm install`
5. 添加环境变量
6. 点击 "Deploy"

## 五、部署到 Railway（连接 GitHub）

### 方式一：Railway CLI
```bash
# 安装 Railway CLI

# 登录
railway login

# 连接项目
railway init
```

### 方式二：Railway 网页

1. 访问 https://railway.app
2. 点击 "New Project"
3. 选择 "Deploy from GitHub repo"
4. 选择 `car-insurance-system`
5. 自动检测并部署
6. 添加环境变量

## 六、GitHub Actions 自动部署

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to Vercel

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - uses: actions/setup-node@v3
        with:
          node-version: '20'
          
      - run: npm ci
      - run: npm run build
      
      - uses: amondnet/vercel-action@v20
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

## 七、常用 Git 命令组合

### 更新代码并重新部署
```bash
# 1. 添加修改
git add .

# 2. 提交
git commit -m "update: 修复bug和优化性能"

# 3. 推送到 GitHub
git push origin main

# 4. Vercel 会自动重新部署
# 或者手动触发部署
vercel --prod
```

### 查看提交历史
```bash
# 简洁历史
git log --oneline

# 图形化历史
git log --graph --pretty=oneline

# 最近5条
git log -5 --pretty=format:"%h - %s, %ar"
```

### 标签管理（版本发布）
```bash
# 创建标签
git tag -a v1.0.0 -m "首次发布"

# 推送标签
git push origin v1.0.0

# 推送所有标签
git push --tags

# 查看标签
git tag -l
```

## 八、故障排查

### 推送被拒绝
```bash
# 强制推送（谨慎使用）
git push -f origin main

# 先拉取再推送
git pull origin main --rebase
git push origin main
```

### 解决冲突
```bash
# 拉取最新代码
git pull origin main

# 手动解决冲突后
git add .
git commit -m "resolve conflict"
git push origin main
```

### 撤销最后一次推送
```bash
# 回退到上一个版本
git reset --soft HEAD~1

# 强制推送
git push -f origin main
```
